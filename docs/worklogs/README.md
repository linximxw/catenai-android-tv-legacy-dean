# 工作日志约定

这个目录用于记录 `catenai-android-tv-legacy-dean` 项目的每日工作日志。

## 命名规则

- 每天一份文件
- 文件名格式：
  - `YYYY-MM-DD-主题.md`

例如：

- `2026-04-13-project-startup-and-day2-log.md`

## 每篇日志至少包含的内容

1. 今天一句话结论
2. 当前阶段说明
3. 今天完成的工作
4. 遇到的问题和处理结果
5. 当前项目状态
6. 下一步计划
7. 阶段提醒

## 记录原则

- 用中文
- 尽量说人话
- 不写空话
- 重点写清楚“为什么这样做”和“现在进行到哪一步”

## 阶段提醒规则

后续执行时遵循下面这个约定：

- 每完成当天工作，补当天工作日志
- 每完成一个明显阶段，单独提醒：
  - 本阶段完成了什么
  - 当前项目到了哪一步
  - 下一阶段准备做什么
